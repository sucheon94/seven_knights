# 세븐나이츠 리버스 길드전 카운터 노트

공유 가능한 정적 웹앱 프로토타입입니다. `index.html`을 브라우저에서 열면 바로 실행됩니다.

## 데이터 수정

기본 데이터는 `data/counters.js`의 `window.SK_COUNTER_DEFAULT_DATA`에 있습니다.

- `heroes`: 영웅 목록
- `type`: `attack`, `magic`, `defense`, `support`, `universal` 중 하나
- `defenseTeams`: 상대 방어팀 조합
- `counters`: 해당 방어팀에 대한 공략덱, 진형, 스킬 순서, 반지, 비고

## 진형

앱 화면에서는 `기본 진형`, `밸런스 진형`, `공격 진형`, `보호 진형` 중 하나를 방어팀/공략덱 각각 따로 선택할 수 있습니다. 각 팀은 5개 위치를 가지고, 그중 최대 3명까지 원하는 칸에 배치합니다.

앱 오른쪽 위 버튼으로 JSON을 내보내거나 가져올 수도 있습니다. 가져온 데이터는 브라우저 `localStorage`에 저장되며, 되돌리기 버튼으로 기본 데이터로 복원할 수 있습니다.

## 엑셀로 공략덱 업데이트

`counter-template.xlsx`가 공략덱 원본 양식입니다.

- `방어팀` 시트: `방어팀ID`, `방어팀명`, 방어 영웅 3명만 입력합니다. 방어팀 위치와 진형은 매칭에 사용하지 않습니다.
- `공략덱` 시트: 방어팀ID별 공략덱, 공략진형, 위치, 스킬순서, 반지, 비고를 입력합니다.
- `참조_영웅` 시트: 앱이 인식하는 영웅 이름 목록입니다. 영웅 이름은 이 시트와 동일하게 입력해야 합니다.
- `사용여부`가 `N`이면 앱에서 제외됩니다.

로컬 서버나 배포 환경에서는 같은 폴더의 `counter-template.xlsx`를 자동으로 읽습니다. `file://`로 직접 열면 브라우저 보안 정책 때문에 자동 읽기가 막히므로, 오른쪽 위 `XL` 버튼으로 엑셀 파일을 선택하면 됩니다.

## 공유 방법

이 폴더를 GitHub Pages, Netlify, Vercel 정적 배포, 또는 개인 웹호스팅에 올리면 됩니다. 서버 코드가 없어서 파일 그대로 배포할 수 있습니다.

## 실제 데이터로 바꿀 때

현재 들어있는 공략 정보는 화면 확인용 샘플입니다. 사용 중인 시트 내용을 `defenseTeams`와 `counters` 구조에 맞게 옮기면 됩니다.
